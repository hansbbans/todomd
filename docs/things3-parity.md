# Things 3 Parity List

## High Impact, Visible Immediately

**1. "This Evening" section in Today**
Things 3 splits Today into two buckets: tasks for the day and tasks for the evening. Users can schedule a task to "This Evening" and it appears in a separate section. TodoMD's Today view has no evening concept.

**2. Separate "Deadline" field from scheduled date**
Things 3 distinguishes *when you want to work on something* (scheduled date / "When") from *when it must be done* (deadline). Deadlines show a yellow/red warning badge as they approach. TodoMD appears to use a single due date for both concepts.

**3. Project progress indicator in sidebar and Browse**
Things 3 shows a subtle circular progress ring or count (`3 of 8`) next to each project showing completion progress. Gives users at-a-glance project health without opening it.

**4. Task count badges on sidebar items**
Things 3 shows badge counts on Inbox (most importantly) and Today in the sidebar. Inbox count is the primary driver for "what needs attention." TodoMD's sidebar appears to show no counts.

**5. Today view header with weekday + date**
Things 3's Today view opens with a large greeting area: the weekday name in large display text, the full date below it. Very calendar-like and grounding. TodoMD has a hero header but it likely just says "Today."

## Core Workflow Features

**6. Headings within projects**
Things 3 lets you add section headings inside a project to organize tasks into groups (e.g., "Phase 1", "Blocked", "Backlog"). This is one of Things 3's most-used features for complex projects. No equivalent appears to exist in TodoMD.

**7. Checklist items (subtasks)**
Things 3 tasks can have a simple checklist of sub-items — not full sub-tasks, just bullet-point checkboxes with labels. These appear inside the expanded task detail. Popular for things like "Pack bag: [ ] passport [ ] charger [ ] adapter."

**8. "Schedule for Today" star vs. due date**
In Things 3, starring a task (or dragging it into Today) schedules it for today without setting a "deadline." This decouples urgency from scheduling. TodoMD seems to use a single due date concept — starring is equivalent to flagging (orange flag), not scheduling.

**9. Logbook organized by completion date**
Things 3's Logbook groups completed tasks by the date they were completed, with date headers ("Yesterday", "Monday", "Last Week"). TodoMD's Logbook likely shows a flat list or simple date metadata.

## Polish and Interaction

**10. Keyboard-to-create (press any key to start a new task)**
In Things 3 on Mac/iPad, pressing any printable key in a list immediately starts creating a new task with that character pre-filled. Eliminates the click → type friction. Very fast for power users.

**11. Magic Plus Button position insertion (iOS)**
In Things 3 on iPhone, you can drag the + button up and down the list before releasing — the insertion point indicator shows where the new task will land. Useful for inserting mid-list without reordering after.

**12. Smoother checkbox animation**
Things 3 has a distinctive checkbox: the circle fills with color, then a checkmark draws itself in, then the row slides out. TodoMD has a completion animation but it may not match this exact sequence.

**13. Cleaner task row — project name placement**
In Things 3, the project/area name appears as a subtle colored dot or short tag below the task title, not as text. The row feels very minimal. TodoMD's metadata line (project, date, tags all on one line) is functional but busier.

**14. Tag filtering pill bar within a view**
Things 3 lets you tap a tag name at the top of a list to filter the current view to just tasks with that tag — no navigation needed. Shows as a row of tappable tag chips below the header. Useful in Anytime/Someday.

**15. Area overview screen**
Things 3's Area view shows all tasks across all projects in that area in one scrollable list, organized by project heading. TodoMD's Area view likely shows a similar grouped list but may not have the same visual hierarchy between projects.

## Nice to Have

**16. "Someday" section within each project**
Things 3 lets you move tasks within a project to a "Someday" section — they stay associated with the project but are deprioritized and hidden from main views. Keeps projects tidy.

**17. Repeating tasks "after completion"**
Things 3 supports two recurrence modes: *fixed interval* (every Monday regardless) and *after completion* (7 days after you check it off). The "after completion" mode is great for habits like "water plants."

**18. Command Bar / keyboard shortcuts panel (Mac)**
Things 3 on Mac has extensive ⌘ shortcuts visible on hover and a searchable command palette. Helps power users stay keyboard-driven.

**19. Drag tasks between views in sidebar**
In Things 3, you can drag a task from the main list and drop it onto a project or area in the sidebar to move it. No context menu required.

**20. Empty state illustrations**
Things 3 has distinctive, minimal empty state graphics (e.g., a small star for an empty Today, a tray for empty Inbox). These reinforce the "done for the day" feeling rather than just showing a blank list.
